▾ How these numbers are sourced — direct vs. derived

When you pick a drive, its MTBF, URE rate, and per-disk IOPS/throughput are used directly in the failure & rebuild model — replacing the generic "HDD / SSD / NVMe" defaults. Per-disk throughput is capped at 4× the type baseline so a single fast NVMe drive doesn't project unrealistic pool-level numbers (controllers, CPU, and ZFS itself become the bottleneck long before raw drive throughput does).

What the datasheets actually publish:

  • MTBF / URE — published for almost all enterprise & NAS drives. Most consumer HDD datasheets (BarraCuda, WD Blue, Toshiba P300) and Samsung consumer SSDs omit URE; the catalog uses conservative class estimates and labels them as such in source comments.
  • SSD / NVMe random IOPS & sequential MB/s — published in every datasheet we cite. Catalog values are taken verbatim for the representative capacity (largest typically; 1 TB SKUs cap lower).
  • HDD sequential MB/s — published; capacity-dependent. Catalog uses the most representative capacity per family.
  • HDD random IOPS — Seagate Exos and WD Ultrastar publish them (4 K QD16 / QD32); catalog uses datasheet figures. NAS, surveillance, and consumer HDDs do not publish random IOPS. For those models the catalog uses ~120 IOPS for 7200 rpm and ~80–100 IOPS for 5400 rpm, derived from average rotational latency + seek time. Treat HDD random IOPS as physics estimates, not vendor numbers.
  • Price/TB — fetched live from every retailer for the selected region (cheapest wins), fallback baked-in USD estimate. Always an estimate, never a quote.
Failure model & references ▾
Disk failure rate (MTBF / AFR)

Per-disk failure follows an exponential model with rate λ = 1 / MTBF. Annual Failure Rate ≈ 8760 / MTBF. Defaults used when you select "Custom drive":

TypeMTBF (h)AFRSource
HDD 1,200,000~0.73 %Seagate Enterprise spec [A1]
SSD 2,000,000~0.44 %Enterprise SSD datasheet typical
NVMe 2,500,000~0.35 %Datacenter NVMe class [C4]

Real-world fleet data tracks somewhat higher: Backblaze reported a 1.36 % annual failure rate across 344k drives in 2025 (down from 1.55 % in 2024). Treat the spec MTBF as a best case. [D1]

Unrecoverable Read Error (URE) risk

Probability of at least one URE while reading B bits during resilver: P = 1 − e−B·r, where r is the per-bit error rate.

ClassURE rate≈ 1 error per
Consumer HDD (e.g. WD Red Plus)1 in 101412.5 TB read
Enterprise / NAS HDD (e.g. Exos, IronWolf Pro)1 in 1015125 TB read
Datacenter SSD1 in 10161.25 PB read
NVMe (consumer / DC)1 in 101712.5 PB read

Manufacturer datasheets quote URE as a worst-case bound, not an average — measured rates are often 10–20× lower in the field. [D2]

Resilver-window data loss

After one disk fails, the vdev can survive up to tol more failures (mirror = N−1, RAID-Zn = n). Probability of exceeding tolerance during resilver window t is approximated by a binomial tail with p = 1 − e−λt per surviving disk. URE risk during the read-every-bit rebuild is added on top.

MTTDL (mean time to data loss) ≈ 1 / (AFRpool · Ploss|failure). This is a model, not a guarantee — correlated failures (same batch, shared PSU, vibration) and silent corruption aren't captured. Always have backups.
